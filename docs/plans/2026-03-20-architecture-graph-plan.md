# Architecture Graph Generator — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Create a bash script that statically analyzes the Orbit Java codebase and generates `architect/architecture-graph.md` — a comprehensive architecture reference for AI agents.

**Architecture:** Single bash script (`scripts/generate-architecture-graph.sh`) using `grep`/`awk`/`sed` to parse Java source files. Output is a Markdown file with Mermaid diagrams. No external dependencies.

**Tech Stack:** Bash, grep, awk, sed (standard Unix tools only)

### Task 1: Script Skeleton + Module Overview

**Files:**
- Create: `scripts/generate-architecture-graph.sh`

**Step 1: Create script with header, variables, and module discovery**

```bash
#!/bin/bash
set -e

# Configuration
SRC_DIR="src/main/java/com/mrbt/orbit"
OUTPUT="architect/architecture-graph.md"
BASE_PKG="com.mrbt.orbit"
TIMESTAMP=$(date -u +"%Y-%m-%dT%H:%M:%SZ")

# Validation
if [ ! -d "$SRC_DIR" ]; then
  echo "Error: $SRC_DIR not found. Run from project root."
  exit 1
fi

# Discover modules (directories with .java files, exclude OrbitApplication.java)
MODULES=()
for dir in "$SRC_DIR"/*/; do
  module=$(basename "$dir")
  MODULES+=("$module")
done

# Counters for summary
TOTAL_ENDPOINTS=0
TOTAL_ENTITIES=0

# Start output
cat > "$OUTPUT" << EOF
<!-- AUTO-GENERATED by scripts/generate-architecture-graph.sh -->
<!-- Do not edit manually. Regenerate with: ./scripts/generate-architecture-graph.sh -->
<!-- Generated: $TIMESTAMP -->

# Orbit Architecture Graph

EOF
```

**Step 2: Add Section 1 — Module Overview & Layer Completeness**

Append to the script a function that checks each module for `api/`, `core/`, `infrastructure/`, `exception/` subdirectories and outputs a Markdown table.

```bash
# --- Section 1: Module Overview ---
echo "## 1. Module Overview & Layer Completeness" >> "$OUTPUT"
echo "" >> "$OUTPUT"
echo "| Module | api | core | infrastructure | exception |" >> "$OUTPUT"
echo "|--------|-----|------|----------------|-----------|" >> "$OUTPUT"

for module in "${MODULES[@]}"; do
  api="—"; core="—"; infra="—"; exc="—"
  [ -d "$SRC_DIR/$module/api" ] && api="✓"
  [ -d "$SRC_DIR/$module/core" ] && core="✓"
  [ -d "$SRC_DIR/$module/infrastructure" ] && infra="✓"
  [ -d "$SRC_DIR/$module/exception" ] && exc="✓"
  echo "| $module | $api | $core | $infra | $exc |" >> "$OUTPUT"
done
echo "" >> "$OUTPUT"
```

### Task 2: Module Dependency Graph (Mermaid)

**Files:**
- Modify: `scripts/generate-architecture-graph.sh`

**Step 1: Add Section 2 — dependency graph analysis**

Parse all `import com.mrbt.orbit.{module}` statements. For each source file, extract which module it belongs to and which other modules it imports. Exclude self-imports and cross-cutting modules (`common`, `config`).

```bash
# --- Section 2: Module Dependency Graph ---
echo "## 2. Module Dependency Graph" >> "$OUTPUT"
echo "" >> "$OUTPUT"
echo '```mermaid' >> "$OUTPUT"
echo "graph TD" >> "$OUTPUT"

# Collect unique edges
declare -A EDGES
for module in "${MODULES[@]}"; do
  # Find all imports from other orbit modules in this module's files
  if [ -d "$SRC_DIR/$module" ]; then
    deps=$(grep -rh "^import ${BASE_PKG}\." "$SRC_DIR/$module" 2>/dev/null \
      | sed "s/import ${BASE_PKG}\.\([a-z_]*\)\..*/\1/" \
      | sort -u)
    for dep in $deps; do
      # Skip self-imports and cross-cutting modules
      if [ "$dep" != "$module" ] && [ "$dep" != "common" ] && [ "$dep" != "config" ]; then
        edge_key="${module}|${dep}"
        if [ -z "${EDGES[$edge_key]+x}" ]; then
          echo "  $module --> $dep" >> "$OUTPUT"
          EDGES[$edge_key]=1
        fi
      fi
    done
  fi
done

echo '```' >> "$OUTPUT"
echo "" >> "$OUTPUT"
```

**Step 2: Run and verify**

Run:
```bash
./scripts/generate-architecture-graph.sh
```

Expected: Mermaid `graph TD` with edges like `payment --> ledger`, `budget --> ledger`, etc. No self-imports, no edges to `common`/`config`.

### Task 3: Cross-cutting Dependencies

**Files:**
- Modify: `scripts/generate-architecture-graph.sh`

**Step 1: Add Section 3 — cross-cutting dependency table**

Same import analysis but only for edges to `common`, `config`, `security`.

```bash
# --- Section 3: Cross-cutting Dependencies ---
echo "## 3. Cross-cutting Dependencies" >> "$OUTPUT"
echo "" >> "$OUTPUT"
echo "| Module | common | config | security |" >> "$OUTPUT"
echo "|--------|--------|--------|----------|" >> "$OUTPUT"

CROSS_MODULES=("common" "config" "security")
for module in "${MODULES[@]}"; do
  # Skip cross-cutting modules themselves
  if [[ " ${CROSS_MODULES[*]} " == *" $module "* ]]; then
    continue
  fi
  row="| $module"
  for cross in "${CROSS_MODULES[@]}"; do
    count=$(grep -rh "^import ${BASE_PKG}\.${cross}\." "$SRC_DIR/$module" 2>/dev/null | wc -l | tr -d ' ')
    if [ "$count" -gt 0 ]; then
      row="$row | ✓ ($count)"
    else
      row="$row | —"
    fi
  done
  echo "$row |" >> "$OUTPUT"
done
echo "" >> "$OUTPUT"
```

### Task 4: Key Classes Per Module

**Files:**
- Modify: `scripts/generate-architecture-graph.sh`

**Step 1: Add Section 4 — classify and list key classes**

Grep for `@RestController`, `@Service`, `@Entity`, `JpaRepository` annotations, extract class names, group by module.

```bash
# --- Section 4: Key Classes Per Module ---
echo "## 4. Key Classes Per Module" >> "$OUTPUT"
echo "" >> "$OUTPUT"

for module in "${MODULES[@]}"; do
  echo "### $module" >> "$OUTPUT"
  echo "" >> "$OUTPUT"

  # Controllers
  controllers=$(grep -rl "@RestController" "$SRC_DIR/$module" 2>/dev/null \
    | xargs -I{} basename {} .java 2>/dev/null | sort)
  if [ -n "$controllers" ]; then
    echo "**Controllers:** $(echo $controllers | sed 's/ /, /g')" >> "$OUTPUT"
  fi

  # Services
  services=$(grep -rl "@Service" "$SRC_DIR/$module" 2>/dev/null \
    | xargs -I{} basename {} .java 2>/dev/null | sort)
  if [ -n "$services" ]; then
    echo "**Services:** $(echo $services | sed 's/ /, /g')" >> "$OUTPUT"
  fi

  # Entities
  entities=$(grep -rl "@Entity" "$SRC_DIR/$module" 2>/dev/null \
    | xargs -I{} basename {} .java 2>/dev/null | sort)
  if [ -n "$entities" ]; then
    echo "**Entities:** $(echo $entities | sed 's/ /, /g')" >> "$OUTPUT"
    TOTAL_ENTITIES=$((TOTAL_ENTITIES + $(echo "$entities" | wc -w)))
  fi

  # Repositories
  repos=$(grep -rl "JpaRepository\|CrudRepository" "$SRC_DIR/$module" 2>/dev/null \
    | xargs -I{} basename {} .java 2>/dev/null | sort)
  if [ -n "$repos" ]; then
    echo "**Repositories:** $(echo $repos | sed 's/ /, /g')" >> "$OUTPUT"
  fi

  echo "" >> "$OUTPUT"
done
```

### Task 5: Database Entity Relationships (Mermaid ER Diagram)

**Files:**
- Modify: `scripts/generate-architecture-graph.sh`

**Step 1: Add Section 5 — parse JPA annotations and generate ER diagram**

For each `@Entity` file:
1. Extract table name from `@Table(name = "...")`
2. Extract relationships from `@ManyToOne`, `@OneToMany`, `@ManyToMany`
3. Extract target entity from field type (e.g., `private AccountEntity account` → `AccountEntity`)
4. Map entity class names to table names
5. Generate Mermaid `erDiagram`

```bash
# --- Section 5: Database Entity Relationships ---
echo "## 5. Database Entity Relationships" >> "$OUTPUT"
echo "" >> "$OUTPUT"
echo '```mermaid' >> "$OUTPUT"
echo "erDiagram" >> "$OUTPUT"

# Build entity-to-table mapping
declare -A ENTITY_TABLE
ENTITY_FILES=$(grep -rl "@Entity" "$SRC_DIR" 2>/dev/null | grep -v "Base\|SoftDeletable\|CreatedOnly")

for file in $ENTITY_FILES; do
  class_name=$(basename "$file" .java)
  table_name=$(grep -oP '@Table\(name\s*=\s*"\K[^"]+' "$file" 2>/dev/null || echo "")
  if [ -n "$table_name" ]; then
    ENTITY_TABLE["$class_name"]="$table_name"
  fi
done

# Parse relationships
declare -A ER_EDGES
for file in $ENTITY_FILES; do
  class_name=$(basename "$file" .java)
  source_table="${ENTITY_TABLE[$class_name]:-}"
  [ -z "$source_table" ] && continue

  # ManyToOne: this table has FK to related table
  m2o_targets=$(grep -B0 -A2 "@ManyToOne" "$file" 2>/dev/null \
    | grep "private " | sed 's/.*private \([A-Za-z]*Entity\).*/\1/' | sort -u)
  for target in $m2o_targets; do
    target_table="${ENTITY_TABLE[$target]:-}"
    [ -z "$target_table" ] && continue
    edge_key="${source_table}|${target_table}|m2o"
    if [ -z "${ER_EDGES[$edge_key]+x}" ]; then
      echo "  $target_table ||--o{ $source_table : \"has\"" >> "$OUTPUT"
      ER_EDGES[$edge_key]=1
    fi
  done

  # ManyToMany
  m2m_targets=$(grep -B0 -A4 "@ManyToMany" "$file" 2>/dev/null \
    | grep "private " | sed 's/.*<\([A-Za-z]*Entity\)>.*/\1/' | sort -u)
  for target in $m2m_targets; do
    target_table="${ENTITY_TABLE[$target]:-}"
    [ -z "$target_table" ] && continue
    edge_key="${source_table}|${target_table}|m2m"
    if [ -z "${ER_EDGES[$edge_key]+x}" ]; then
      echo "  $source_table }o--o{ $target_table : \"many-to-many\"" >> "$OUTPUT"
      ER_EDGES[$edge_key]=1
    fi
  done
done

echo '```' >> "$OUTPUT"
echo "" >> "$OUTPUT"
```

### Task 6: API Endpoint Inventory

**Files:**
- Modify: `scripts/generate-architecture-graph.sh`

**Step 1: Add Section 6 — parse controller annotations for endpoint inventory**

For each `@RestController` file:
1. Extract base path from `@RequestMapping("...")`
2. Extract method mappings: `@GetMapping`, `@PostMapping`, `@PatchMapping`, `@DeleteMapping`
3. Extract summary from `@Operation(summary = "...")`
4. Combine into table grouped by module

```bash
# --- Section 6: API Endpoint Inventory ---
echo "## 6. API Endpoint Inventory" >> "$OUTPUT"
echo "" >> "$OUTPUT"

CONTROLLER_FILES=$(grep -rl "@RestController" "$SRC_DIR" 2>/dev/null)

for file in $CONTROLLER_FILES; do
  # Determine module
  module=$(echo "$file" | sed "s|$SRC_DIR/\([^/]*\)/.*|\1|")
  class_name=$(basename "$file" .java)

  # Extract base path from @RequestMapping
  base_path=$(grep -oP '@RequestMapping\("\K[^"]+' "$file" 2>/dev/null || echo "")

  echo "### $module — $class_name" >> "$OUTPUT"
  echo "" >> "$OUTPUT"
  echo "| Method | Path | Description |" >> "$OUTPUT"
  echo "|--------|------|-------------|" >> "$OUTPUT"

  # Process each HTTP method mapping
  # Use awk to correlate mapping annotations with @Operation summary
  awk '
    /@(Get|Post|Put|Delete|Patch)Mapping/ {
      # Extract method
      match($0, /@(Get|Post|Put|Delete|Patch)Mapping/, m)
      method = toupper(m[1])

      # Extract path (may be empty for root)
      path = ""
      if (match($0, /Mapping\("([^"]*)"/, p)) {
        path = p[1]
      } else if (match($0, /Mapping\(value\s*=\s*"([^"]*)"/, p)) {
        path = p[1]
      }
      full_path = "'"$base_path"'" path

      # Look ahead for @Operation summary
      summary = ""
      getline next_line
      if (match(next_line, /summary\s*=\s*"([^"]*)"/, s)) {
        summary = s[1]
      }

      printf "| %s | `%s` | %s |\n", method, full_path, summary
      endpoint_count++
    }
  ' "$file" >> "$OUTPUT"

  # Count endpoints in this file
  file_endpoints=$(grep -cE "@(Get|Post|Put|Delete|Patch)Mapping" "$file" 2>/dev/null || echo "0")
  TOTAL_ENDPOINTS=$((TOTAL_ENDPOINTS + file_endpoints))

  echo "" >> "$OUTPUT"
done
```

Note: The awk approach reads `@Operation` on the line before or after the mapping annotation. Since in this codebase `@Operation` comes on the line _after_ `@*Mapping` (verified in `PaymentMethodController.java`), we read the next line for summary.
