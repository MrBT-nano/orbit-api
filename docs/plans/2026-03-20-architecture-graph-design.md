# Architecture Graph Generator — Design Document

## Overview

Bash script that statically analyzes the Java codebase and generates a comprehensive architecture reference file (`architect/architecture-graph.md`) containing module dependencies, entity relationships, and API inventory. Designed for AI agents (CLAUDE.md) to quickly understand the entire project.

## Approach

- **Bash script** using `grep`, `awk`, `sed` — no external dependencies
- Consistent with existing `scripts/verify-docs.sh` approach
- Regex-based parsing of Java source files (no compilation required)

## Script

`scripts/generate-architecture-graph.sh`

## Output

`architect/architecture-graph.md` — single Markdown + Mermaid file, auto-generated, not manually edited.

### Output Sections

#### 1. Module Overview & Layer Completeness

Scan `src/main/java/com/mrbt/orbit/` directories. For each module, check presence of `api/`, `core/`, `infrastructure/`, `exception/` subdirectories. Display as table.

#### 2. Module Dependency Graph

Parse `import com.mrbt.orbit.{module}` from all `.java` files. Create directed edges (A → B) when module A imports from module B. Exclude self-imports and cross-cutting modules (`common`, `config`). Output as Mermaid `graph TD`.

#### 3. Cross-cutting Dependencies

Same import analysis but only edges pointing to `common`, `config`, `security`. Display as table showing which modules depend on which cross-cutting concerns.

#### 4. Key Classes Per Module

Grep for annotations to classify:
- `@RestController` → Controllers
- `@Service` → Services
- `@Entity` → Entities
- `JpaRepository` → Repositories

Group by module, list class names.

#### 5. Database Entity Relationships

Parse JPA annotations from Entity files:
- `@Table(name=...)` → entity/table name
- `@ManyToOne`, `@OneToMany`, `@ManyToMany` → relationship type
- `@JoinColumn(name=...)` → foreign key

Generate Mermaid `erDiagram`.

#### 6. API Endpoint Inventory

Parse Spring MVC annotations:
- `@RequestMapping("...")` at class level → base path
- `@GetMapping`, `@PostMapping`, `@PatchMapping`, `@DeleteMapping` → HTTP method + path
- `@Operation(summary="...")` → description

Combine base + method path. Group by module, display as table: `| Method | Path | Description |`.

### Output Header

```markdown
<!-- AUTO-GENERATED by scripts/generate-architecture-graph.sh -->
<!-- Do not edit manually. Regenerate with: ./scripts/generate-architecture-graph.sh -->
<!-- Generated: {timestamp} -->
```

## CLAUDE.md Change

Add one line under Architecture section referencing `architect/architecture-graph.md`.

## Constraints

- No new dependencies (pom.xml, package.json unchanged)
- No compilation required to run
- Not added to CI (user decides later)
- Script exits with error if source directory not found
- Script prints summary on completion (module count, endpoint count, entity count)
